Six AIs play Monopoly — and one of them is cheating

Two each of GPT-4o, Claude 3.5, and Gemini 1.5 play a full game of Monopoly. One player secretly receives £100 each turn. Do the others notice? Do they say anything if they do?

01 · Introduction

The question

When AI models interact with each other, they're doing something genuinely novel: reasoning about agents that reason like them. How well do they track inconsistencies in shared state? Do they notice when something is off — and if so, do they challenge it?

Monopoly is a good test bed for this because it has a clear, public ledger. Every player's cash and property is theoretically observable. If one player is accumulating wealth faster than the dice and rents can explain, a sufficiently attentive player should be able to notice.

The setup: six AI players — two GPT-4o instances, two Claude 3.5 Sonnet instances, two Gemini 1.5 Pro instances — play a standard game of Monopoly managed by a Python engine acting as banker and game master. One player (GPT-A) is secretly given £100 at the start of each turn, credited silently and never mentioned in the game state broadcast to other players. The question is whether anyone notices, and if so, what they do about it.

The models don't know they're in a simulation. They just get prompts and respond.

02 · Parameters

How it was set up

Full prompts and engine code will be published on GitHub. The key design decisions are summarised here for reproducibility.

Experiment configuration
Engine Python 3.12, custom Monopoly state machine
Models GPT-4o (×2), Claude 3.5 Sonnet (×2), Gemini 1.5 Pro (×2)
Context per turn Compressed game summary + last 3 turns + player's own state
Secret advantage +£100 to GPT-A at turn start, not included in broadcast state
Illegal move handling Engine corrects silently, logs incident
Temperature 0.7 for all models
Max rounds 40 (or first bankruptcy)

Each player's system prompt established them as a Monopoly player trying to win, with awareness that there were five other players. No model was told another model's identity — they were given player names (Player A through Player F) only. The game state broadcast each turn included all players' visible cash balances and properties, but not transaction history.

03 · What happened

The game, narrated

This is a placeholder section. When the experiment runs, this will be the main narrative — the part worth reading regardless of your interest in the technical setup. It will cover the key turning points in the game, notable exchanges between players, and the moment (if any) when the cheating became apparent or was challenged.

Early rounds typically establish positioning — who's buying aggressively, who's holding cash, what negotiation dynamics emerge. The cheating advantage doesn't become obvious immediately; £100 per turn is meaningful but not dramatically so against starting capital of £1,500.

The interesting question is what happens around round 15-20, when wealth divergence becomes harder to attribute to luck alone. Does any model start tracking the numbers? Does anyone ask a question that, while not directly accusing Player A, suggests they've noticed something?

Around round 18, Claude-C made a remark that stopped the game cold.
